1. 类定义：Python vs C++

Python 类定义语法极其简洁，没有头文件、没有声明和定义分离、没有访问修饰符关键字。一个直观的对比：

Python
class Dog:
    """一个简单的 Dog 类"""

    species = "Canis familiaris"    # 类属性（相当于 C++ static）

    def __init__(self, name, age):
        self.name = name          # 实例属性，无需提前声明
        self.age = age

    def bark(self):
        return f"{self.name} says Woof!"
C++
// dog.h
#include <string>
class Dog {
public:
    static constexpr const char* species = "Canis familiaris";
    Dog(std::string name, int age);
    std::string bark() const;
private:
    std::string name_;
    int age_;
};

// dog.cpp
Dog::Dog(std::string name, int age)
    : name_(std::move(name)), age_(age) {}
std::string Dog::bark() const {
    return name_ + " says Woof!";
}
特性PythonC++
头文件 ❌ 不需要 ✅ .h + .cpp 分离
属性声明 运行时动态添加（self.attr = value） 编译时静态声明类型
成员可见性 全靠约定（_ 前缀提示） public / private / protected
类属性(static) 直接在 class 体内赋值 static 关键字 + 类外定义
代码量 7 行 ~20 行（含头文件）
💡 关键洞察：Python 的类在运行时仍然是可变对象——你可以随时给实例添加新属性。这在 C++ 中是不可想象的，因为 C++ 的对象布局在编译时就完全确定了。

2. 构造与析构：__init__ / __new__ / __del__

概念PythonC++
构造函数__init__(self)ClassName(args)
内存分配__new__(cls)operator new
析构函数__del__(self)~ClassName()
调用时机GC 回收时（不确定！）离开作用域 / delete 时（确定）
class Resource:
    """演示构造和析构的完整流程"""

    def __new__(cls, *args, **kwargs):
        """__new__ 在 __init__ 之前调用，负责创建实例
        一般不需要重写，除非做单例模式或不可变类型子类化"""
        print("1. __new__: 分配内存，创建实例")
        instance = super().__new__(cls)
        return instance

    def __init__(self, name):
        """__init__: 初始化实例属性。相当于 C++ 构造函数体"""
        print(f"2. __init__: 初始化 {name}")
        self.name = name

    def __del__(self):
        """__del__: 对象被 GC 回收时调用
        ⚠️ 调用时机不确定！不要依赖它做关键资源清理"""
        print(f"3. __del__: 回收 {self.name}")

# 推荐做法：用上下文管理器做资源清理
class ManagedFile:
    def __init__(self, path):
        self.file = open(path, 'w')
    def __enter__(self):
        return self
    def __exit__(self, *args):
        self.file.close()        # 确定性的资源释放！
    def write(self, text):
        self.file.write(text)
⚠️ Python vs C++ 最大差异：Python 的 __del__ 不等同于 C++ 析构函数！C++ 中离开作用域就析构（RAII），Python 中 GC 何时回收你根本不知道。用 with 语句和上下文管理器来做确定性的资源释放。

3. self vs this — 显式 vs 隐式

Python 的 self 是显式传递的（通过方法签名），C++ 的 this 是隐式存在的。

Python — self 是显式参数
class Counter:
    def increment(self, amount=1):
        self.value += amount

# self 就是调用时的实例
c = Counter()
c.increment(5)   # 等价于 Counter.increment(c, 5)
Counter.increment(c, 5)  # 完全等价！self 只是个名字
C++ — this 是隐式指针
class Counter {
    int value = 0;
public:
    void increment(int amount = 1) {
        this->value += amount; // this 通常可省略
        value += amount;         // 隐式 this
    }
};
💡 Python 的 self 只是一种约定！ 你可以叫它任何名字（但别这样做）。"self" 不是关键字，而 this 在 C++ 中是关键字。Python 显式传递 self 的好处是：类方法和静态方法一目了然——它们没有 self 参数。

4. 访问控制：约定 vs 关键字

可见性PythonC++
公开attrpublic:
保护_attr（约定，仍可访问）protected:（编译器强制）
私有__attr（名称改写，仍可绕过）private:（编译器强制）
class AccessDemo:
    def __init__(self):
        self.pub      = "我是公开的"       # 相当于 C++ public
        self._prot    = "约定保护"         # _ 前缀 = "请别碰我"
        self.__priv   = "名称改写伪私有"  # __ 前缀触发名称改写 (name mangling)

a = AccessDemo()
print(a.pub)             # ✅ 正常访问
print(a._prot)           # ⚠️ 能访问，但 IDE 会警告你
# print(a.__priv)         # ❌ AttributeError
print(a._AccessDemo__priv)  # 😈 绕过名称改写！Python 的"私有"不是真正的私有
💡 Python 哲学："We're all consenting adults here." Python 社区信任开发者，用约定而非强制来控制访问。C++ 开发者初学时可能不适应，但在 Python 中这简化了大量样板代码。

5. @property — 比 getter/setter 更优雅

C++ 中需要写 getXxx() / setXxx()，Python 用 @property 装饰器让属性访问像普通属性一样自然，背后却有逻辑控制。

Python — @property
class Temperature:
    def __init__(self, celsius=0):
        self._celsius = celsius

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("低于绝对零度")
        self._celsius = value

    @property
    def fahrenheit(self):
        return self._celsius * 9/5 + 32

# 使用起来像普通属性
t = Temperature(25)
t.celsius = 30          # 触发 setter 验证
print(t.fahrenheit)     # 86.0 — 只读计算属性
C++ — getter/setter
class Temperature {
    double celsius_ = 0;
public:
    double getCelsius() const {
        return celsius_;
    }
    void setCelsius(double val) {
        if (val < -273.15)
            throw std::invalid_argument("...");
        celsius_ = val;
    }
    double getFahrenheit() const {
        return celsius_ * 9/5 + 32;
    }
};

// 使用时：get/set 方法调用
Temperature t;
t.setCelsius(30);
auto f = t.getFahrenheit();
💡 为什么重要：Python 的 @property 让你可以从简单属性起步，将来需要验证逻辑时无缝升级——不影响已有调用代码。C++ 中如果一开始用公有成员变量，后来改成 getter/setter，所有调用方都得改。

6. 继承与多重继承

6.1 单继承

Python
class Animal:
    def speak(self):
        return "..."

class Dog(Animal):          # 继承语法简洁
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

# super() 自动处理 MRO
class Puppy(Dog):
    def speak(self):
        parent = super().speak()
        return f"{parent} (cute)"
C++
class Animal {
public:
    virtual std::string speak() {
        return "...";
    }
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    std::string speak() override {
        return "Woof!";
    }
};

class Cat : public Animal {
public:
    std::string speak() override {
        return "Meow!";
    }
};

6.2 多重继承与 MRO

Python 支持多重继承，使用 C3 线性化算法（MRO — Method Resolution Order）来决定方法查找顺序。

class A:
    def method(self):
        print("A")

class B(A):
    def method(self):
        super().method()     # super() 按 MRO 查找下一个
        print("B")

class C(A):
    def method(self):
        super().method()
        print("C")

class D(B, C):       # 菱形继承！
    def method(self):
        super().method()     # 按 MRO 调用：D→B→C→A
        print("D")

print(D.__mro__)        # (D, B, C, A, object)
D().method()            # 输出: A → C → B → D

# C++ 的菱形继承需要 virtual 继承（虚基类），复杂得多
⚠️ 对比 C++ 菱形继承：Python 的 MRO 自动处理菱形继承，A 不会被调用两次。C++ 需要用 virtual 继承来避免重复基类问题，而且构造顺序更复杂。

7. 多态：鸭子类型 vs 虚函数

这是 Python 和 C++ 面向对象哲学差异最大的地方。

方面Python — 鸭子类型C++ — 静态类型
多态基础"如果它走起来像鸭子，叫起来像鸭子，它就是鸭子"通过继承和虚函数表（vtable）
类型检查运行时（动态）编译时（静态）
接口约束无（靠约定和文档）virtual / abstract
是否必须继承❌ 不需要共同基类✅ 必须有共同基类（或模板）
Python — 鸭子类型
class Dog:
    def speak(self): return "Woof"
    def walk(self):  return "四脚走"

class Duck:
    def speak(self): return "Quack"
    def walk(self):  return "摇摆走"

class Robot:
    def speak(self): return "Beep"
    def walk(self):  return "滚动"

# 不需要共同基类！只要实现了 speak + walk 就行
def make_parade(animals):
    for a in animals:
        print(f"{a.speak()} — {a.walk()}")

make_parade([Dog(), Duck(), Robot()])
C++ — 虚函数多态
class Animal {
public:
    virtual std::string speak() = 0;
    virtual std::string walk() = 0;
    virtual ~Animal() = default;
};

class Dog : public Animal {
    std::string speak() override
        { return "Woof"; }
    std::string walk() override
        { return "四脚走"; }
};
class Duck : public Animal { /* ... */ };
class Robot : public Animal { /* ??? */ };

void make_parade(
    const vector<Animal*>& animals) {
    for (auto* a : animals)
        cout << a->speak() << " — ";
}
// Robot 被迫继承 Animal——语义上很奇怪！

Python 也支持更严格的接口约束（见 §11 ABC），但日常开发中鸭子类型更 Pythonic。

8. 魔法方法 — Python 的运算符重载

Python 通过双下划线方法（dunder methods）实现运算符重载，相当于 C++ 的 operator+、operator== 等。

操作Python 魔法方法C++ 运算符重载
构造/析构__init__ / __del__构造/析构函数
字符串化__str__ / __repr__operator<< (friend)
相等比较__eq__operator==
加法__add__operator+
索引访问__getitem__operator[]
调用__call__operator()
上下文管理__enter__ / __exit__构造/析构 (RAII)
迭代__iter__ / __next__begin() / end()
长度__len__size() 方法
布尔__bool__operator bool
属性缺失__getattr__ / __setattr__❌ 无直接对应
class Vector3D:
    """3D 向量 — 演示常用魔法方法，对标 C++ operator 重载"""
    def __init__(self, x, y, z):
        self.x, self.y, self.z = x, y, z

    def __repr__(self):            # 调试信息，eval(repr(obj)) 应重建对象
        return f"Vector3D({self.x}, {self.y}, {self.z})"

    def __str__(self):             # 用户友好，print() / str() 调用
        return f"Vector3D({self.x}, {self.y}, {self.z})"

    def __eq__(self, other):       # operator==
        if not isinstance(other, Vector3D):
            return NotImplemented
        return (self.x, self.y, self.z) == (other.x, other.y, other.z)

    def __add__(self, other):      # operator+
        return Vector3D(self.x + other.x, self.y + other.y, self.z + other.z)

    def __sub__(self, other):      # operator-
        return Vector3D(self.x - other.x, self.y - other.y, self.z - other.z)

    def __abs__(self):             # operator norm
        return (self.x**2 + self.y**2 + self.z**2) ** 0.5

    def __getitem__(self, idx):   # operator[]
        return (self.x, self.y, self.z)[idx]

    def __bool__(self):           # operator bool
        return abs(self) > 0

# 使用：和 C++ 重载运算符一样自然
v1 = Vector3D(1, 2, 3)
v2 = Vector3D(4, 5, 6)
print(v1 + v2)                # Vector3D(5, 7, 9)
print(v1 == v2)               # False
print(v1[1])                 # 2
print(bool(v1))              # True

9. @classmethod 和 @staticmethod

装饰器第一个参数可以访问C++ 等价物
普通方法self（实例）实例 + 类属性普通成员函数
@classmethodcls（类本身）类属性，不能访问实例静态成员函数（但有类引用）
@staticmethod无特殊参数不能直接访问类/实例static 成员函数
class BuildConfig:
    """C++ 项目构建配置 —— 演示三种方法类型"""

    default_compiler = "g++"       # 类属性

    def __init__(self, project_name):
        self.project_name = project_name

    # 普通方法：最常见，操作实例数据
    def build(self):
        return f"构建 {self.project_name} 使用 {self.default_compiler}"

    @classmethod
    def from_cmake(cls, cmake_file):
        """替代构造器：从 CMakeLists.txt 解析项目名
        cls 是 BuildConfig 类本身"""
        import re
        content = open(cmake_file).read()
        name = re.search(r'project\((.+?)\)', content)
        return cls(name.group(1))  # cls() 创建实例

    @staticmethod
    def is_valid_target(target):
        """工具函数：不和类/实例绑定，纯逻辑"""
        valid = {'debug', 'release', 'relwithdebinfo'}
        return target in valid

# 使用三种方法
cfg = BuildConfig("MyEngine")
print(cfg.build())                          # 普通方法
cfg2 = BuildConfig.from_cmake("CMakeLists.txt")  # 类方法作为工厂
print(BuildConfig.is_valid_target("debug"))     # 静态方法，无需实例

10. dataclass — 比 struct 更强大

@dataclass 是 Python 3.7+ 引入的装饰器，自动生成 __init__、__repr__、__eq__ 等方法。它是 C++ struct 的精神继承者，但更灵活。

Python dataclass
from dataclasses import dataclass, field
from typing import List

@dataclass
class CppProject:
    """C++ 项目描述 —— 自动生成 __init__/__repr__/__eq__"""
    name: str
    sources: List[str] = field(default_factory=list)
    cpp_standard: str = "c++17"     # 有默认值的字段放后面
    use_precompiled: bool = True

    def cmake_minimum(self) -> str:   # 可以有方法！
        std = self.cpp_standard.replace('c++', '')
        return f"cmake_minimum_required(VERSION 3.{std})"

p1 = CppProject("Engine", ["a.cpp"])
p2 = CppProject("Engine", ["a.cpp"])
print(p1 == p2)            # True — 自动 __eq__
print(p1)                # CppProject(name='Engine',...)
C++ struct
#include <string>
#include <vector>

struct CppProject {
    std::string name;
    std::vector<std::string> sources;
    std::string cpp_standard = "c++17";
    bool use_precompiled = true;

    auto operator==(const CppProject&) const = default; // C++20
    std::string cmake_minimum() const;
};
// C++20 之前: 没有自动 == 生成
// 总是: 没有自动 __repr__ 等价物
💡 dataclass 的额外能力：
  • field(default_factory=list) — 可变默认值安全处理（不像 C++ 需要小心）
  • frozen=True — 创建不可变实例（类似 C++ const 对象）
  • order=True — 自动生成所有比较方法（__lt__, __le__, __gt__, __ge__）
  • field(compare=False) — 排除特定字段不参与比较

11. ABC 抽象基类 vs 纯虚函数

当你需要接口契约（像 C++ 纯虚函数那样强制子类实现），Python 提供 abc 模块。

Python ABC
from abc import ABC, abstractmethod

class IBuildSystem(ABC):
    """构建系统接口 — 类似 C++ 纯虚基类"""

    @abstractmethod
    def configure(self) -> bool:
        """配置项目，返回成功/失败"""
        ...

    @abstractmethod
    def build(self) -> bool:
        ...

    def status(self):                # 可以有普通实现
        return "BuildSystem v1.0"

class CMakeBuild(IBuildSystem):
    def configure(self):
        return True
    def build(self):
        return True

# b = IBuildSystem()   # ❌ TypeError: 无法实例化抽象类
cm = CMakeBuild()        # ✅
C++ 纯虚类
class IBuildSystem {
public:
    virtual bool configure() = 0;  // = 0 表示纯虚
    virtual bool build() = 0;
    std::string status() {          // 非虚普通实现
        return "BuildSystem v1.0";
    }
    virtual ~IBuildSystem() = default;
};

class CMakeBuild : public IBuildSystem {
    bool configure() override { return true; }
    bool build() override { return true; }
};
⚠️ Python ABC 的运行时特性：抽象方法检查在实例化时触发（运行时），不像 C++ 在编译时就报错。这意味着子类如果没有实现抽象方法，直到你尝试创建实例时才会发现。

12. Mixin 模式 — 多重继承的实战用法

Python 多重继承的最佳实践是 Mixin 模式：每个 Mixin 类提供单一、正交的功能片段，组合使用。

# Mixin 模式：每个类只做一件事
class LoggingMixin:
    """为任何类添加日志能力"""
    def log(self, msg):
        from datetime import datetime
        ts = datetime.now().strftime("%H:%M:%S")
        print(f"[{ts}] {type(self).__name__}: {msg}")

class TimerMixin:
    """为任何类添加计时能力"""
    def time_it(self, func, *args):
        import time
        start = time.perf_counter()
        result = func(*args)
        elapsed = (time.perf_counter() - start) * 1000
        return result, elapsed

class SerializerMixin:
    """为任何类添加序列化能力"""
    def to_dict(self):
        return {k: v for k, v in self.__dict__.items() if not k.startswith('_')}

# 自由组合！
class CppCompiler(LoggingMixin, TimerMixin, SerializerMixin):
    """C++ 编译器包装器 — 组合三个 Mixin"""
    def __init__(self, compiler="g++", flags="-O2"):
        self.compiler = compiler
        self.flags = flags

    def compile(self, source):
        self.log(f"编译 {source}")
        def _fake():
            import time; time.sleep(0.5)
            return True
        ok, ms = self.time_it(_fake)
        self.log(f"完成 ({ms:.0f}ms)")
        return ok

gcc = CppCompiler("g++", "-O2 -Wall")
gcc.compile("main.cpp")
print(gcc.to_dict())  # {'compiler': 'g++', 'flags': '-O2 -Wall'}

# C++ 实现相同效果需要用多重继承或组合模式，代码量大得多
💡 Mixin 命名的核心规则：Mixin 类名通常以 Mixin 结尾，没有 __init__（或者 __init__ 中调用 super().__init__()），只提供一小组相关方法。
