1. 模块基础：import 的多种姿势

在 C++ 中，我们用 #include 引入头文件，用 namespace 组织代码。Python 的 import 同时承担了这两项职责，而且更灵活。

C++ — #include + namespace
// mylib.h
#pragma once
namespace mylib {
    int add(int a, int b);
    constexpr int VERSION = 2;
}

// main.cpp
#include "mylib.h"
using namespace mylib; // 引入整个命名空间
int main() {
    int r = add(1, 2);
    return 0;
}
Python — import 多种写法
# mylib.py
def add(a, b):
    return a + b

VERSION = 2

# main.py
import mylib                    # 导入整个模块
r = mylib.add(1, 2)

from mylib import add           # 只导入特定名称
r = add(1, 2)

from mylib import add as plus  # 别名
r = plus(1, 2)

from mylib import *           # 不推荐！

核心区别

特性C++ #includePython import
机制预编译期文本替换运行时加载模块对象
重复包含需 #pragma once / include guard自动缓存，只加载一次
选择性导入不支持，只能全部引入支持 from X import Y
别名namespace aliasimport as
循环引用编译错误运行时可能出问题（需注意）
注意：避免使用 from module import *，它会污染命名空间，且无法追踪名称来源，相当于 C++ 中的 using namespace std; 在头文件中滥用。

2. 模块搜索路径与 __name__

Python 查找模块时，按以下顺序搜索路径（类似 C++ 的 include path 但更灵活）：

import sys
print(sys.path)
# 输出类似：
# ['',                          # 当前目录（脚本所在目录）
#  'C:\\Python312\\Lib',          # 标准库
#  'C:\\Python312\\DLLs',
#  'C:\\Users\\you\\AppData\\...\\site-packages']  # 第三方包

# 添加自定义搜索路径（类似 C++ 的 -I 选项）
sys.path.append('D:/my_cpp_tools/python')

__name__ 与模块测试

这是 C++ 没有的概念：每个 .py 文件都能判断自己是被导入还是直接运行。

C++ — 分离测试与库
// mylib.cpp — 必须单独写测试文件
// test_mylib.cpp
#include "mylib.h"
#include <cassert>
int main() {
    assert(add(1,2) == 3);
}
Python — 同文件内测试
# mylib.py
def add(a, b):
    return a + b

# 直接运行时执行，被导入时跳过
if __name__ == "__main__":
    assert add(1, 2) == 3
    print("All tests passed!")
技巧：Python 模块被首次导入时会从头执行一遍（只一次），模块对象被缓存在 sys.modules 中。后续 import 直接返回缓存对象，类似 C++ 的 include guard 效果。

3. 包（Package）与 __init__.py

Python 的"包"对应 C++ 项目中的目录结构 + CMake 组织方式，但更轻量。

目录结构对比

C++ 项目结构
my_project/
├── CMakeLists.txt
├── include/
│   └── myproject/
│       ├── core.h
│       └── utils.h
├── src/
│   ├── core.cpp
│   └── utils.cpp
└── tests/
    └── test_core.cpp
Python 包结构
my_project/
├── pyproject.toml       # 类似 CMakeLists.txt
├── myproject/           # 包目录
│   ├── __init__.py      # 包标识 + 初始化
│   ├── core.py
│   └── utils.py
└── tests/
    └── test_core.py

__init__.py 的作用

# myproject/__init__.py
# 1. 标记目录为 Python 包（Python 3.3+ 可省略，但建议保留）
# 2. 包级初始化代码
# 3. 控制 from myproject import * 的导出内容

from .core import App, Config   # 简化外部导入路径
from .utils import helpers

__version__ = "1.0.0"
__all__ = ["App", "Config", "helpers"]  # import * 时只导出这些

# 外部使用：
# from myproject import App  （而不是 from myproject.core import App）
类比：__init__.py 类似 C++ 头文件中的"聚合头文件"（如 #include "myproject/all.h"），它把子模块的公开 API 重新导出，简化用户导入路径。

4. 相对导入 vs 绝对导入

C++ — 包含路径
// 绝对路径（推荐）
#include "myproject/core.h"

// 相对路径（脆弱）
#include "../core.h"  // 不推荐

// C++ 没有真正的"相对包含"
// 依赖 -I 编译选项
Python — 导入方式
# 绝对导入（推荐）
from myproject.core import App

# 相对导入（包内部使用）
from .core import App      # 同级
from ..utils import foo   # 上级
from . import core         # 导入同级模块

# 相对导入只在包内部有效
# 直接运行含相对导入的脚本会报错！
常见坑：直接运行包内的 .py 文件（如 python myproject/core.py），相对导入会报 ImportError。必须用 python -m myproject.core 或在包外运行。

5. 标准库精选：C++ 开发者必备工具

Python 标准库是它的"超能力"之一 — C++ 需要 Boost/第三方库才能实现的功能，Python 往往内置了。

5.1 os / pathlib — 文件系统操作

C++17 filesystem
#include <filesystem>
#include <iostream>
namespace fs = std::filesystem;

// 遍历目录
for (auto& entry : fs::directory_iterator("src")) {
    std::cout << entry.path() << "\n";
}

// 路径拼接
auto p = fs::path("src") / "main.cpp";

// 检查是否存在
bool exists = fs::exists(p);
Python pathlib
from pathlib import Path

# 遍历目录
for entry in Path("src").iterdir():
    print(entry)

# 路径拼接（用 / 运算符！）
p = Path("src") / "main.cpp"

# 检查是否存在
exists = p.exists()

# 递归查找所有 .cpp 文件
cpps = list(Path("src").rglob("*.cpp"))
推荐：优先使用 pathlib 而非 os.path。pathlib 的 / 运算符拼接路径比 os.path.join() 更直观，且跨平台。

5.2 json — 配置文件处理

import json

# 读取 JSON 配置（C++ 项目常用场景）
with open("compile_commands.json") as f:
    data = json.load(f)    # 直接解析为 Python 对象

# 查找所有 .cpp 编译命令
cpp_entries = [e for e in data if e["file"].endswith(".cpp")]

# 写入 JSON
config = {"compiler": "clang++", "flags": ["-std=c++20", "-O2"]}
with open("config.json", "w") as f:
    json.dump(config, f, indent=2)

5.3 collections — 高级数据结构

from collections import defaultdict, Counter, namedtuple

# defaultdict — 不用检查 key 是否存在（类似 C++ map[]）
groups = defaultdict(list)
for name, ext in [("main", ".cpp"), ("utils", ".h"), ("core", ".cpp")]:
    groups[ext].append(name)
# {".cpp": ["main", "core"], ".h": ["utils"]}

# Counter — 统计频率
word_counts = Counter(["error", "warning", "error", "error"])
print(word_counts.most_common(1))  # [("error", 3)]

# namedtuple — 轻量级结构体（类似 C++ struct）
CompileResult = namedtuple("CompileResult", ["success", "output", "errors"])
result = CompileResult(True, "obj/main.o", [])
print(result.success)  # 按名称访问

5.4 argparse — 命令行参数解析

C++ 手动解析或用第三方库
// C++ 没有内置参数解析
// 常用：CLI11, cxxopts, getopt
#include <CLI/CLI.hpp>
CLI::App app{"Build tool"};
int jobs = 4;
app.add_option("-j,--jobs", jobs);
app.parse(argc, argv);
Python argparse（内置）
import argparse

parser = argparse.ArgumentParser(description="Build tool")
parser.add_argument("-j", "--jobs", type=int, default=4)
parser.add_argument("targets", nargs="*")
args = parser.parse_args()

print(f"Building with {args.jobs} jobs")
print(f"Targets: {args.targets}")

6. 第三方包管理：pip 与虚拟环境

C++ 用 vcpkg/conan 管理依赖，Python 用 pip。但 Python 还需要"虚拟环境"来隔离项目依赖 — 这在 C++ 中不太常见（C++ 通过不同 build 目录实现类似效果）。

虚拟环境

# 创建虚拟环境（类似 C++ 的 out-of-source build 目录）
python -m venv .venv

# 激活（Windows）
.venv\Scripts\activate

# 激活（Linux/Mac）
source .venv/bin/activate

# 安装包
pip install pybind11 cmake-format

# 导出依赖清单（类似 CMake 的 find_package 列表）
pip freeze > requirements.txt

# 在新环境还原
pip install -r requirements.txt
最佳实践：每个 Python 项目都应使用虚拟环境。将 .venv/ 加入 .gitignore，但提交 requirements.txt。

7. 实战：C++ 项目的 Python 工具模块

下面是一个实际可用的工具模块结构，用于辅助 C++ 项目开发：

# cppdev/  — 包目录
# ├── __init__.py
# ├── scanner.py     — 扫描 C++ 源文件
# ├── builder.py     — 构建辅助
# └── analyzer.py    — 代码分析

# cppdev/__init__.py
"""C++ Development Helper Tools."""
from .scanner import scan_sources
from .builder import build_project

__version__ = "0.1.0"
__all__ = ["scan_sources", "build_project"]

# cppdev/scanner.py
"""Scan C++ source files in a project."""
from pathlib import Path
from collections import defaultdict

def scan_sources(root: str | Path) -> dict:
    """扫描目录下的 C++ 源文件，按类型分组。"""
    root = Path(root)
    groups = defaultdict(list)

    for ext in ["*.cpp", "*.h", "*.hpp", "*.cc"]:
        for f in root.rglob(ext):
            groups[ext.lstrip("*")].append(str(f))

    return dict(groups)

# 使用：
# from cppdev import scan_sources
# files = scan_sources("D:/my_project/src")
# print(files[".cpp"])  # 所有 .cpp 文件列表
练习
  1. 创建一个 cppdev 包，包含 __init__.py 和至少两个子模块
  2. 实现 scan_sources()，扫描指定目录下的 C++ 文件并按扩展名分组
  3. 用 __name__ == "__main__" 为每个模块添加自测代码
  4. 使用 argparse 让工具支持命令行调用
  5. 创建 requirements.txt（即使暂时无第三方依赖也建议创建）
