1. 文件操作基础 — open / read / write

Python 的文件操作比 C++ 简洁一个量级。核心差异：Python 用 with 语句自动管理资源，不需要手写 close()。

C++ — 读取文件 
// C++: 需要显式 open/close，逐行读取繁琐
std::ifstream file("input.txt");
if (!file.is_open()) {
    throw std::runtime_error("Cannot open file");
}
std::string line;
std::vector<std::string> lines;
while (std::getline(file, line)) {
    lines.push_back(line);
}
file.close();  // 必须手动关闭
std::cout << lines.size() << " lines";
Python — 读取文件 
with open("input.txt") as file:
    lines = file.readlines()  # 读所有行

print(f"{len(lines)} lines")
# with 块结束后自动关闭，无需 close()

文件模式速查

模式含义C++ 对应
"r"只读（默认）std::ios::in
"w"写入（覆盖，不存在则创建）std::ios::out | std::ios::trunc
"a"追加写入std::ios::out | std::ios::app
"x"排他创建（文件存在则失败）需手动检查
"r+"读写（不截断）std::ios::in | std::ios::out
"b"二进制模式（可与上述组合）std::ios::binary
💡 Python 的"安全写入"模式
Python 的 "x" 模式很实用——只有文件不存在时才创建，防止意外覆盖。C++ 需要先用 std::filesystem::exists 检查，Python 一行搞定。

逐行读取与文件迭代

C++ — 逐行处理 
// C++: 读取→操作→另存
std::ifstream in("source.cpp");
std::ofstream out("output.txt");
std::string line;
int count = 0;
while (std::getline(in, line)) {
    if (line.find("//") != std::string::npos) {
        ++count;
    }
}
std::cout << count << " comments";
Python — 文件迭代器 
with open("source.cpp") as f:
    # 文件对象本身就是迭代器！
    comments = sum(1 for line in f if "//" in line)
print(f"{comments} comments")
📌 内存对比
file.readlines() 把整个文件读入内存（适合小文件），for line in file 是惰性的（适合大文件）。Python 提供了写 C++ 时梦寐以求的简洁性。

2. 路径处理 — os.path vs pathlib

Python 提供了两套路径处理方案。旧式 os.path 沿用至今，而 Python 3.4+ 的 pathlib 用面向对象的方式处理路径——就像 C++17 的 std::filesystem。

C++17 — filesystem 
namespace fs = std::filesystem;
fs::path p = fs::current_path() / "src" / "main.cpp";
std::cout << p.string();      // 路径字符串
std::cout << p.filename();   // main.cpp
std::cout << p.parent_path();// src/
std::cout << p.extension();  // .cpp
std::cout << p.stem();       // main
Python — pathlib 
from pathlib import Path

p = Path.cwd() / "src" / "main.cpp"
print(p)              # D:\proj\src\main.cpp
print(p.name)         # main.cpp
print(p.parent)       # D:\proj\src
print(p.suffix)       # .cpp
print(p.stem)         # main

pathlib 常用操作

Python — pathlib 实用操作 
from pathlib import Path

src = Path("src/")

# 遍历所有 .cpp 文件（递归）
cpp_files = src.rglob("*.cpp")

# 过滤 + 操作
for f in src.glob("*.h"):
    print(f.stat().st_size)  # 文件大小
    print(f.read_text())      # 直接读文本（不用 open!）

# 创建目录（相当于 mkdir -p）
Path("build/obj/x64").mkdir(parents=True, exist_ok=True)

# 拼接路径（/ 运算符重载！）
config_path = Path.home() / ".config" / "myapp.ini"
💡 选 pathlib，忘掉 os.path
pathlib 是 Python 的 std::filesystem——面向对象、直观、没有字符串拼接陷阱。在新项目中，没有理由再用 os.path 了。

3. 数据序列化 — json、csv、struct

C++ 序列化通常要引入第三方库（如 nlohmann/json、protobuf），而 Python 标准库直接提供 json、csv、pickle、struct。对于构建脚本和配置管理，json 是首选。

JSON — C++ 生态的痛点，Python 的舒适区

C++ — nlohmann/json 
// 需要引入第三方库
#include <nlohmann/json.hpp>
using json = nlohmann::json;

// 构建 JSON
json config = {
    {"name", "MyApp"},
    {"version", "1.0"},
    {"cxx_standard", 17}
};

// 序列化 + 写文件
std::ofstream o("config.json");
o << config.dump(2);
Python — 内置 json 
import json

# 构建 JSON
config = {
    "name": "MyApp",
    "version": "1.0",
    "cxx_standard": 17,
}

# 序列化 + 写文件，一行搞定
with open("config.json", "w") as f:
    json.dump(config, f, indent=2)

CSV 操作

Python — csv 模块 
import csv

# 写入 CSV
with open("targets.csv", "w", newline="") as f:

struct — 二进制打包

Python 的 struct 模块让你像 C/C++ struct 一样打包/解包二进制数据——对于解析自定义二进制格式非常有用：

C++ — 写入二进制 
struct Header {
    uint32_t magic = 0xDEADBEEF;
    uint16_t version = 1;
    float scale = 1.5f;
};
// memcpy → fwrite → close
Python — struct 打包 
import struct

# I=uint32, H=uint16, f=float32
data = struct.pack("IHf", 0xDEADBEEF, 1, 1.5)
# data = b'\xef\xbe\xad\xde\x01\x00\x00\x00\xc0?'

with open("header.bin", "wb") as f:
    f.write(data)

4. 异常处理 — try/except/else/finally

Python 的异常机制与 C++ 有相似之处，但多了 else 和 finally 的灵活组合，而且 Python 鼓励 EAFP（Easier to Ask Forgiveness than Permission）风格。

C++ — try/catch 
try {
    auto data = load_file(filepath);
    process(data);
} catch (const std::ios_base::failure& e) {
    std::cerr << "IO Error: " << e.what();
} catch (const std::exception& e) {
    std::cerr << "Error: " << e.what();
}
// 没有 else/finally 语法糖
Python — try/except/else/finally 
try:
    data = load_file(filepath)
    process(data)
except FileNotFoundError:
    print("文件不存在")
except PermissionError:
    print("没有权限")
else:
    # 这里只有 try 成功时执行！
    print("处理成功")
finally:
    # 无论如何都执行（清理资源）
    cleanup()
try 块 → 成功 → else 块 → finally 块
try 块 → 异常 → except 块 → finally 块
try 块 → 异常 → 未匹配 → 向上传播

常见内置异常

Python 异常触发场景C++ 对应
FileNotFoundError文件不存在std::ios_base::failure
PermissionError权限不足需检查 errno
ValueError类型对但值不对std::invalid_argument
TypeError类型错误编译期错误（C++ 不会触发）
KeyError字典键不存在无（未定义行为/返回 end iterator）
IndexError列表索引越界std::out_of_range
ZeroDivisionError除以零未定义行为（整数）/ inf（浮点）
OSError系统调用失败需检查 errno / system_error
⚠️ EAFP vs LBYL
LBYL (Look Before You Leap)：先检查再行动。C++ 风格。
EAFP (Easier to Ask Forgiveness than Permission)：先尝试，捕获异常。Python 风格。

在 Python 中，try/except 比预检查（if key in dict）更 Pythonic，也是性能更优的路径（正常情况零开销）。

5. 自定义异常 — raise 与异常链

Python 中定义异常超简单，而且支持 from 关键字建立异常链——比 C++ 的 nested_exception 直观得多。

C++ — 自定义异常 
class ParseError : public std::exception {
    std::string msg;
public:
    ParseError(const std::string& file, int line)
        : msg("Parse error in " + file
              + ":" + std::to_string(line)) {}
    const char* what() const noexcept override {
        return msg.c_str();
    }
};
throw ParseError("config.xml", 42);
Python — 自定义异常 
# 一行定义异常类！
class ParseError(Exception):
    pass

# 抛异常
raise ParseError(f"Parse error in config.xml:42")

# 带 from 的异常链（Python 独有！）
try:
    raw = open("config.xml")
except OSError as e:
    raise ParseError("配置加载失败") from e
💡 raise from 的妙用
raise X from Y 建立了一条清晰的因果链。traceback 会显示 "The above exception was the direct cause of the following exception"。C++ 的 throw_with_nested 概念类似但使用繁琐。

异常层次结构

Python — 组织异常层次 
class BuildError(Exception):
    """所有构建相关异常的基类"""
    pass

class CompileError(BuildError):
    """编译阶段错误"""
    pass

class LinkError(BuildError):
    """链接阶段错误"""
    pass

# 统一的异常处理
try:
    build()
except BuildError as e:  # 捕获所有构建错误
    print(f"构建失败: {e}")
except Exception as e:
    print(f"意外错误: {e}")

6. 上下文管理器 — with 语句的秘密

如果你来自 C++，你爱 RAII（Resource Acquisition Is Initialization）。Python 的上下文管理器提供了类似的安全感，但机制是显式的。

核心关联：C++ RAII ≈ Python 上下文管理器（with 语句）

with 是如何工作的

C++ — RAII 
// 构造函数获取资源
// 析构函数自动释放
{
    std::lock_guard<std::mutex> lock(mtx);
    // 临界区代码
    unsafe_operation();
}   // lock 在此自动释放

// 即使异常也保证析构
Python — with 语句 
with open("data.txt") as f:
    data = f.read()
    process(data)
# f 在此自动关闭

# 等价于：
f = open("data.txt")
try:
    data = f.read()
    process(data)
finally:
    f.close()  # 即使异常也保证执行

实现自己的上下文管理器

两种方式：基于类（__enter__/__exit__）和基于生成器（contextlib）。后者通常更简洁：

基于类 — __enter__/__exit__ 
class WorkingDir:
    """临时切换工作目录（自动恢复）"""
    def __init__(self, path):
        self.path = path
        self.old = None

    def __enter__(self):
        self.old = Path.cwd()
        os.chdir(self.path)
        return self

    def __exit__(self, *args):
        os.chdir(self.old)

with WorkingDir("build/"):
    subprocess.run(["cmake", ".."])
# 自动回到原目录，安全！
基于生成器 — contextmanager 
from contextlib import contextmanager

@contextmanager
def working_dir(path):
    old = Path.cwd()
    os.chdir(path)
    try:
        yield  # 这是切入点
    finally:
        os.chdir(old)  # 保证恢复

with working_dir("build/"):
    subprocess.run(["cmake", ".."])

实用上下文管理器场景

场景C++ RAIIPython 上下文管理器

7. 实战：C++ 项目文件扫描器

综合运用今天学到的所有知识，构建一个实用的 C++ 项目文件扫描工具。它能：

  • 递归扫描目录，找出所有 .cpp/.h/.hpp 文件
  • 统计每个文件的代码行数（区分代码/空行/注释）
  • 分析 #include 依赖关系
  • 生成 JSON 格式的分析报告
  • 优雅的错误处理（跳过无权限目录等）
cpp_project_scanner.py — 核心逻辑 
class CppProjectScanner:
    """C++ 项目文件扫描分析器"""

    def scan(self, root: Path) -> list[FileInfo]:
        """递归扫描所有 C++ 源文件"""
        files = []
        for pattern in ["*.cpp", "*.cc", "*.cxx",
                        "*.h", "*.hpp", "*.hxx"]:
            try:
                for filepath in root.rglob(pattern):
                    try:
                        info = self._analyze_file(filepath)
                        files.append(info)
                    except PermissionError:
                        continue  # 跳过无权限文件
            except OSError:
                continue  # 跳过无法访问的目录
        return files

    def _analyze_file(self, filepath: Path) -> FileInfo:
        """分析单个文件：行数、注释、include"""
        with open(filepath, errors="replace") as f:
            lines = f.readlines()

        code_lines = blank_lines = comment_lines = 0
        includes = []
        in_block_comment = False

        for line in lines:
            stripped = line.strip()
            if not stripped:
                blank_lines += 1
            elif stripped.startswith("//"):
                comment_lines += 1
            else:
                code_lines += 1
            # 提取 #include
            if stripped.startswith("#include"):
                includes.append(stripped)

        return FileInfo(path=filepath, code=code_lines,
                          blank=blank_lines, comment=comment_lines,
                          includes=includes)
🚀 运行方式
# 分析当前目录
python cpp_project_scanner.py .

# 分析指定项目 + 输出 JSON 报告
python cpp_project_scanner.py D:\MyCppProject -o report.json

# 使用示例演示
python cpp_project_scanner.py --demo

⚠️ 今天最重要的三个注意事项
  1. 总是用 with 打开文件 — Python 没有析构函数的确定性保证（GC 回收时间不确定），不用 with 可能造成文件句柄泄漏
  2. 异常不要"裸捕获" — except: 会捕获 KeyboardInterrupt 等系统异常。至少用 except Exception:
  3. pathlib 优于字符串拼接 — os.path.join 和手动 "/" 拼接都容易出错，Path / "subdir" / "file" 安全又直观
🔗 EAFP vs LBYL — Python 的异常哲学
在 C++ 中，大量使用异常是昂贵的，所以 LBYL（先检查再行动）是主流。Python 正好相反：try/except 的正常路径开销极低，EAFP（先行动再处理异常）是 Pythonic 的正确姿势。
