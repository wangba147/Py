1. 为什么 C++ 开发者要学 Python 正则

如果你是 C++ 程序员，你大概率对 std::regex 印象不好：API 难用、性能玄学、连个简单的分组都要写一堆代码。

但日常 C++ 开发中，文本处理需求无处不在：

  • 解析编译器的 error: 'foo' was not declared in this scope 错误日志
  • 从成百上千个源文件里提取 TODO / FIXME 注释
  • 批量重命名变量、函数、类（重构辅助）
  • 解析 CMake 输出、JSON 配置文件、protobuf 生成代码
  • 把 Windows 路径 C:\foo\bar 转成 Unix 风格

这些活儿用 C++ 写一个能跑的版本可能要半天；用 Python 通常 5~10 行搞定。

C++ — std::regex
// 匹配 4 位数字年份（看着就累）
std::regex re(R"((\d{4}))");
std::string s = "created in 2024";
std::smatch m;
if (std::regex_search(s, m, re)) {
    int year = std::stoi(m[1].str());
    std::cout << year << std::endl;
}
Python — re
import re
s = "created in 2024"
m = re.search(r'(\d{4})', s)
if m:
    year = int(m.group(1))
    print(year)   # 2024
核心优势：Python 的 re 模块是 C++ 生态里 std::regex、Boost.Regex、Qt QRegularExpression、ICU 之中最容易上手、表达力最强的方案。配合 pathlib、subprocess，能瞬间搭建一个文本处理工具链。

2. re 模块基础：search / match / findall / sub

Python 的 re 提供四个高频函数，覆盖 90% 的日常场景：

函数作用返回
re.search(pat, s)在 s 中找第一个匹配Match 对象 / None
re.match(pat, s)只在 s 的开头匹配Match 对象 / None
re.findall(pat, s)返回所有匹配的列表list[str] / list[tuple]
re.finditer(pat, s)返回所有匹配的迭代器迭代 Match 对象
re.sub(pat, repl, s)替换新字符串
re.split(pat, s)按模式切分list[str]
import re

s = "My phone: 138-1234-5678, office: 010-8765-4321"

# 1. search: 找第一个
m = re.search(r'\d{3}-\d{4}-\d{4}', s)
print(m.group())          # 138-1234-5678

# 2. match: 必须在开头
m2 = re.match(r'My', s)
print(m2.group())         # My  (如果改成 Phone 就匹配不到)

# 3. findall: 全部提取
phones = re.findall(r'\d{3}-\d{4}-\d{4}', s)
print(phones)                # ['138-1234-5678', '010-8765-4321']

# 4. sub: 替换
masked = re.sub(r'\d{4}', '****', s)
print(masked)                # My phone: 138-****-****, ...
search vs match 易混淆点：match 只检查字符串开头，不匹配会返回 None；想找任意位置用 search；想匹配整个字符串用 fullmatch。

正则的"语法糖"很少，但每一种都值得记住。下面这张表是 C++ 开发者最常查的：

Python 原始字符串 r"..."：写正则强烈建议用 r'...' 前缀，避免 \d 被 Python 字符串解析成退格符。r'\d' 就是两个字符 \d，'\d' 才是 \\d（反斜杠+d）。 
# 错误：'\\d' 实际上是 '\\d'，匹配反斜杠后跟 d
re.search('\d{4}', '2024')   # ❌ 匹配不到

# 正确：r 前缀保留原始字符
re.search(r'\d{4}', '2024')  # ✅ 2024

4. 分组捕获与命名分组

用 ( ) 括起来的部分会被"捕获"——这在解析复杂文本时至关重要。

4.1 匿名分组

# 解析日期 YYYY-MM-DD
date = "2024-03-15"
m = re.search(r'(\d{4})-(\d{2})-(\d{2})', date)
if m:
    year, month, day = m.group(1), m.group(2), m.group(3)
    print(f"{year}/{month}/{day}")   # 2024/03/15

# group(0) 是整个匹配
print(m.group(0))   # 2024-03-15

4.2 命名分组 — 强烈推荐

用 (?P<name>...) 给分组起名字，可读性大幅提升，也方便后期维护。

C++ — 难用
// std::regex 不支持命名分组
// 只能靠索引硬记 m[1], m[2]...
std::regex re(R"((\d{4})-(\d{2})-(\d{2}))");
// 谁是月？谁是日？靠注释
Python — 优雅
m = re.search(
    r'(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})',
    "2024-03-15"
)
print(m.group('year'))    # 2024

4.3 反向引用（高级）

命名/匿名分组还可以在同一个正则内部用 \1 或 (?P=name) 复用——常用于匹配重复模式（如 HTML 标签、引号配对）。

# 匹配成对的双引号字符串
re.findall(r'"([^"]*)"', '"hello", "world"')
# ['hello', 'world']

# 匹配对称的 HTML 标签（开放 = 关闭）
re.search(r'<(?P<tag>\w+)>.*?</(?P=tag)>',
         "<div>hello</div>")   # ✅ 匹配
re.search(r'<(?P<tag>\w+)>.*?</(?P=tag)>',
         "<div>hello</span>")  # ❌ 不匹配

5. 贪婪 vs 非贪婪

Python 的量词默认是贪婪（greedy）的——会尽量匹配最长的字符串。加上 ? 后变成非贪婪（lazy）——匹配最短。

C++ 默认行为
// std::regex 也是贪婪
// 但语法 .*? 同样支持
Python 示例
html = "<b>foo</b><b>bar</b>"

# 贪婪：.* 匹配到最远
re.findall(r'<b>.*</b>', html)
# ['<b>foo</b><b>bar</b>']

# 非贪婪：.*? 尽快停下
re.findall(r'<b>.*?</b>', html)
# ['<b>foo</b>', '<b>bar</b>']
经验法则：解析 HTML/XML 这类嵌套结构时，几乎一定要用非贪婪。解析日志、CSV 这种行式数据时，贪婪更省事。

6. 编译正则：re.compile 的性能优势

在循环里反复用同一个正则时，一定要先编译——这是把 Python 文本处理从"能跑"提升到"快跑"的关键。

C++ std::regex
// C++ 思路：把 regex 对象放循环外
std::regex re(R"(\d+)");
for (auto& line : lines) {
    std::smatch m;
    std::regex_search(line, m, re);
}
Python re.compile
pattern = re.compile(r'\d+')
for line in lines:
    m = pattern.search(line)
    # 比 re.search(r'\d+', line) 快 5~10 倍

编译对象的所有方法

import re

p = re.compile(r'\d+')
p.search(s)         # 等价 re.search
p.match(s)          # 等价 re.match
p.findall(s)        # 等价 re.findall
p.finditer(s)       # 等价 re.finditer
p.sub(repl, s)      # 等价 re.sub
p.pattern            # 原字符串 '\d+'
p.flags              # 编译时设置的标志位

7. 标志位：re.IGNORECASE / MULTILINE / DOTALL

标志简写作用
re.IGNORECASEre.I忽略大小写
re.MULTILINEre.M^ $ 匹配每行的开头/结尾
re.DOTALLre.S. 也匹配换行符
re.ASCIIre.A\w \d \s 仅匹配 ASCII
re.VERBOSEre.X允许多行 + 注释（强烈推荐用于复杂正则）

re.VERBOSE：写出可读的正则

phone_re = re.compile(r'''
    ^                          # 开头
    (?P<area>\d{3})            # 区号 3 位
    -                          # 分隔符
    (?P<num>\d{4})             # 前 4 位
    -                          # 分隔符
    (?P<end>\d{4})             # 后 4 位
    $                          # 结尾
''', re.VERBOSE)

m = phone_re.search("138-1234-5678")
print(m.groupdict())
# {'area': '138', 'num': '1234', 'end': '5678'}
组合标志：re.compile(pat, re.I | re.M)。位运算 | 是 Python 标志位的标准用法，区别于 C++ 的 std::regex::icase 等枚举。

8. 字符串方法：split / strip / replace / join

不是所有文本处理都要正则。Python 的字符串自带了一组"瑞士军刀"：

方法作用示例
s.split(sep)切分"a,b,c".split(",") → ['a','b','c']
s.strip()去两端空白" hi\n".strip() → "hi"
s.lstrip() / rstrip()去左/右空白—
s.replace(old, new)替换"a-b".replace("-", "_")
sep.join(list)拼接",".join(['a','b']) → "a,b"
s.startswith(p)前缀判断"CMake".startswith("CM")
s.endswith(p)后缀判断"main.cpp".endswith(".cpp")
s.upper() / lower()大小写—
s.find(sub)找子串位置—
C++ 字符串处理
// 找子串 + 切分要写一堆代码
auto pos = s.find(",");
auto first = s.substr(0, pos);
auto rest  = s.substr(pos + 1);

// split 没有标准库，要么手写循环
// 要么用 boost::split、absl::StrSplit
Python 一行解决
first, rest = s.split(",", 1)
parts = s.split(",")   # 全部切分
clean = s.strip()
upper = s.upper()
何时用正则、何时用字符串方法：能用 str.split/str.replace/str.startswith 解决的就别用正则——可读性、性能、可维护性都更好。只在"模糊匹配"或"复杂模式"时才上 re。

9. 实战一：解析 C++ 编译错误日志

在 CI（GitHub Actions / Jenkins）里跑构建时，常常需要把编译器的错误提取出来，汇总成报告。下面是 GCC 风格错误的解析：

# 模拟一段 GCC 错误输出
log = """
main.cpp: In function 'int main()':
main.cpp:12:5: error: 'foo' was not declared in this scope
   12 |     foo();
      |     ^~~
main.cpp:15:9: warning: unused variable 'x' [-Wunused-variable]
   15 |     int x = 42;
      |         ^
utils.cpp:42:1: error: expected '}' at end of input
   42 |
      | ^
"""

# 模式: 文件:行:列: 严重等级: 消息
pattern = re.compile(
    r'^(?P<file>[\w./\\-]+):(?P<line>\d+):(?P<col>\d+):\s*'
    r'(?P<severity>error|warning|note):\s*(?P<msg>.*)$',
    re.MULTILINE
)

issues = []
for m in pattern.finditer(log):
    issues.append(m.groupdict())

for i in issues:
    print(f"[{i['severity'].upper():7}] {i['file']}:{i['line']}: {i['msg']}")

# 输出:
# [ERROR  ] main.cpp:12: 'foo' was not declared in this scope
# [WARNING] main.cpp:15: unused variable 'x' [-Wunused-variable]
# [ERROR  ] utils.cpp:42: expected '}' at end of input
练习：改写上面的代码，把结果按"文件"分组，统计每个文件有多少个 error / warning，并按 error 数倒序排序（提示：collections.Counter + sorted）。

10. 实战二：从源码中提取 TODO/FIXME

代码审查时经常需要找：

  • TODO — 待办
  • FIXME — 需要修复
  • XXX — 危险
  • HACK — 临时方案
  • NOTE — 提醒
from pathlib import Path
import re

# 匹配: // TODO(username): 2024-03-15: message
# 或:   /* FIXME: ... */
todo_re = re.compile(r'''
    (?P<tag>TODO|FIXME|XXX|HACK|NOTE)         # 标签
    (?:\((?P<owner>[\w]+)\))?                # 可选负责人
    (?::\s*(?P<date>\d{4}-\d{2}-\d{2}))?      # 可选日期
    \s*:\s*                                  # 分隔
    (?P<msg>.*?)                              # 消息（非贪婪）
    \s*(?:\*/|$)                             // 结尾
''', re.VERBOSE | re.IGNORECASE)

def scan_todos(root: Path):
    items = []
    for cpp_file in root.rglob("*.cpp"):
        try:
            text = cpp_file.read_text(encoding='utf-8')
        except UnicodeDecodeError:
            continue
        for i, line in enumerate(text.splitlines(), 1):
            for m in todo_re.finditer(line):
                items.append({
                    'file': str(cpp_file),
                    'line': i,
                    **m.groupdict()
                })
    return items

# 使用
todos = scan_todos(Path("./src"))
for t in todos:
    owner = t['owner'] or "unassigned"
    print(f"{t['file']}:{t['line']:3} {t['tag']:6} ({owner}) {t['msg']}")

11. 实战三：批量重命名/格式化代码

重构 C++ 项目时，批量替换类名、函数名是常态。下面是一个"安全替换"工具——只替换完整单词，避免误伤 MyClass 里的 Class：

from pathlib import Path
import re

def safe_replace(
    root: Path,
    old: str,
    new: str,
    extensions=".cpp .hpp .h .cc".split()
):
    """
    把文件中所有完整单词 old 替换为 new。
    \b 是单词边界，避免误伤子串。
    """
    pattern = re.compile(rf'\b{re.escape(old)}\b')

    stats = {'scanned': 0, 'modified': 0, 'replacements': 0}

    for ext in extensions:
        for f in root.rglob(f"*{ext}"):
            stats['scanned'] += 1
            text = f.read_text(encoding='utf-8')
            new_text, n = pattern.subn(new, text)
            if n > 0:
                f.write_text(new_text, encoding='utf-8')
                stats['modified'] += 1
                stats['replacements'] += n
                print(f"  {f}: {n} 处替换")

    return stats

# 使用
result = safe_replace(
    Path("./src"),
    old="OldName",
    new="NewName"
)
print(f"\n扫描 {result['scanned']} 文件，修改 {result['modified']} 个，"
      f"共 {result['replacements']} 处替换。")
小心正则特殊字符：old 是用户输入时，必须用 re.escape(old) 转义，否则 old = "A.B" 会变成"匹配任意字符"。\b...\b 是单词边界，避免 foo 替换 foobar。

  2. 把一个 C++ 源文件中的所有 // ... 单行注释和 /* ... */ 多行注释替换为 /* ... */ 形式（提示：用 re.sub + 回调函数）。
