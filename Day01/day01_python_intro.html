1. 为什么 C++ 开发者需要 Python？

如果你每天都在写 C++，可能会觉得 Python「太慢」「不够底层」。但在实际开发中，Python 是 C++ 开发者的最佳搭档：

场景C++Python
构建脚本 CMake / Makefile 更灵活的脚本控制
代码生成 手写或模板元编程 脚本批量生成 .h/.cpp
自动化测试 Google Test + 手动配置 pytest 快速编写测试
数据分析 需自己实现 NumPy / Pandas 开箱即用
调试辅助 GDB 命令行 写脚本解析日志、分析 Core Dump
原型验证 编译 → 链接 → 运行 即写即运行，快速验证思路
核心理念：Python 不替代 C++，而是补位。用 Python 处理「胶水工作」，让 C++ 专注在性能关键路径上。

3. 语法差异速览

作为 C++ 开发者，最大的思维转变不是学新语法，而是忘掉一些 C++ 的习惯。

3.1 无分号、无大括号

C++
if (x > 0) {
    std::cout << "positive"
              << std::endl;
}
Python
if x > 0:
    print("positive")
🔑 Python 用缩进表示代码块，不用 {}。缩进是语法的一部分，不是风格选择。建议统一使用 4 个空格。

3.2 动态类型 vs 静态类型

C++
int x = 42;          // 必须声明类型
std::string s = "hi"; // 类型固定
// x = "hello";       // 编译错误！
Python
x = 42          # 无需声明类型
s = "hi"        # 自动推断
x = "hello"     # 完全合法！类型可以变
实用提示：Python 3.5+ 支持 Type Hints（类型注解），可以写出类似 C++ 的类型约束：
x: int = 42，但注解不会在运行时强制检查，它是给工具（mypy、IDE）用的。

3.3 注释风格

C++
// 单行注释
/* 多行
   注释 */
Python
# 单行注释（Python只有这一种）
"""
多行字符串，常被当作
多行注释使用
"""

3.4 函数定义

C++
int add(int a, int b) {
    return a + b;
}

// 默认参数
void greet(std::string name = "World") {
    std::cout << "Hello, " << name;
}
Python
def add(a: int, b: int) -> int:
    return a + b

# 默认参数
def greet(name: str = "World") -> None:
    print(f"Hello, {name}")

3.5 Main 函数

C++
int main(int argc, char* argv[]) {
    // 程序入口
    return 0;
}
Python
def main():
    # 程序入口
    pass

if __name__ == "__main__":
    main()
为什么用 if __name__ == "__main__"？
这样当文件被其他模块 import 时，main 不会自动执行。类似 C++ 中只有包含 main 的 .cpp 才是程序入口。

4.2 列表（对应 vector）

C++
std::vector<int> nums = {1, 2, 3};
nums.push_back(4);
int x = nums[0];    // 1
nums.size();         // 4
// 遍历
for (int n : nums) {
    std::cout << n;
}
Python
nums = [1, 2, 3]
nums.append(4)
x = nums[0]        # 1
len(nums)          # 4
# 遍历
for n in nums:
    print(n)

4.3 字典（对应 map）

C++
std::map<std::string, int> m;
m["age"] = 25;
if (m.count("age")) {
    std::cout << m["age"];
}
Python
m = {"age": 25}
m["age"]           # 25
# 安全访问
m.get("name", "N/A")  # "N/A"
if "age" in m:
    print(m["age"])

4.4 字符串操作

Python 字符串操作远比 C++ 简洁，这是日常开发中高频使用的能力：

s = "Hello, World!"

# 常用操作
s.lower()          # "hello, world!"    — C++: std::transform
s.upper()          # "HELLO, WORLD!"
s.split(", ")      # ["Hello", "World!"] — C++: 手动分割
",".join(["a","b"])# "a,b"              — C++: 手动拼接
s.replace("World", "Python")  # "Hello, Python!"
f"pi = {3.14:.2f}" # "pi = 3.14"        — f-string 格式化
s[7:12]            # "World"            — 切片，C++: substr
len(s)             # 13

4.5 Python 独有：列表推导式

这是 Python 最强大的特性之一，用一行代码完成 C++ 需要多行循环的工作：

C++
std::vector<int> squares;
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) {
        squares.push_back(i * i);
    }
}
// squares = {0, 4, 16, 36, 64}
Python
squares = [i * i for i in range(10) if i % 2 == 0]
# [0, 4, 16, 36, 64]

5. 第一个实用脚本：C++ 头文件生成器

来写一个真正有用的 Python 脚本——它可以根据类名自动生成 C++ 的 .h 和 .cpp 模板文件。这是 Python 辅助 C++ 开发的典型场景。

"""
cpp_gen.py — C++ 头文件/源文件生成器
用法: python cpp_gen.py MyClass
会在当前目录生成 MyClass.h 和 MyClass.cpp
"""

import sys
import os

def generate_header(class_name: str) -> str:
    """生成 .h 文件内容"""
    guard = f"{class_name.upper()}_H_"
    return f"""#ifndef {guard}
#define {guard}

class {class_name} {{
public:
    {class_name}();
    ~{class_name}();

private:
    // TODO: 成员变量
}};

#endif // {guard}
"""

def generate_source(class_name: str) -> str:
    """生成 .cpp 文件内容"""
    return f"""#include "{class_name}.h"

{class_name}::{class_name}() {{
    // TODO: 构造函数实现
}}

{class_name}::~{class_name}() {{
    // TODO: 析构函数实现
}}
"""

def main():
    if len(sys.argv) < 2:
        print(f"用法: python {sys.argv[0]} ClassName")
        sys.exit(1)

    class_name = sys.argv[1]

    # 写入文件
    with open(f"{class_name}.h", "w", encoding="utf-8") as f:
        f.write(generate_header(class_name))
    print(f"✓ 已生成 {class_name}.h")

    with open(f"{class_name}.cpp", "w", encoding="utf-8") as f:
        f.write(generate_source(class_name))
    print(f"✓ 已生成 {class_name}.cpp")

if __name__ == "__main__":
    main()

运行方式：

python cpp_gen.py Player
# 输出:
# ✓ 已生成 Player.h
# ✓ 已生成 Player.cpp
这个脚本展示了什么？
① sys.argv 接收命令行参数（类似 C++ 的 argc/argv）
② f-string 模板生成代码（比 C++ 的字符串拼接优雅太多）
③ with open() 自动管理文件资源（类似 C++ 的 RAII）
④ 类型注解 class_name: str 和 -> str 提升可读性
