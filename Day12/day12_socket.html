概念1. Socket 是什么？

Socket（套接字）是操作系统提供的网络通信端点。你可以把它理解成一个「网络版的文件描述符」——

  • 客户端通过 socket 连接到服务器
  • 服务器通过 socket 监听并接受连接
  • 连接建立后，双方通过 send()/recv() 收发数据
如果你写过 C++ 的 socket 编程（<sys/socket.h> 或 Winsock2），会发现 Python 的 socket 模块就是同样的 API 去掉繁琐的错误码处理。连函数名都一模一样！

实战2. TCP 客户端

TCP 客户端三步走：创建 socket → 连接服务器 → 收发数据。

Python — TCP 客户端
import socket

# 1. 创建 TCP socket（SOCK_STREAM 表示 TCP）
sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# 2. 连接服务器
sock.connect(("example.com", 80))

# 3. 发送 HTTP 请求
sock.sendall(b"GET / HTTP/1.1\r\nHost: example.com\r\n\r\n")

# 4. 接收响应
response = sock.recv(4096)
print(response.decode())

sock.close()
命令行 — 最简测试
$ python -c "import socket; s=socket.socket(); s.connect(('example.com',80)); \
  s.sendall(b'GET / HTTP/1.1\r\nHost: example.com\r\n\r\n'); print(s.recv(1024).decode())"
C++ 对比（Linux/Winsock）
// 需要初始化、错误检查、清理
int sock = socket(AF_INET, SOCK_STREAM, 0);
struct sockaddr_in addr{};
addr.sin_family = AF_INET;
addr.sin_port = htons(80);

const char* req = "GET / HTTP/1.1\r\nHost: ...\r\n\r\n";

recv(sock, buf, sizeof(buf), 0);
close(sock);  // 或 closesocket(sock)
Python 等价写法
import socket

sock = socket.socket(socket.AF_INET,
                      socket.SOCK_STREAM)
sock.connect(("example.com", 80))
sock.sendall(b"GET / HTTP/1.1\r\n"
              b"Host: example.com\r\n"
              b"\r\n")
response = sock.recv(4096)
print(response.decode())
sock.close()
Python 的 send() 可能不会一次性发送所有数据（和 C++ 一样）。生产代码务必使用 sendall() 或用循环确保发送完整。

实战3. TCP 服务器

服务器端多一步：socket → bind → listen → accept。每次 accept 返回一个新的 socket 用于和该客户端通信。

Python — 回显服务器（Echo Server）
import socket

# 1. 创建 socket
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)

# 2. 绑定地址和端口
server.bind(("0.0.0.0", 8888))

# 3. 开始监听（backlog = 5）
server.listen(5)
print("[*] 监听 0.0.0.0:8888 ...")

while True:
    # 4. 接受连接（阻塞直到有客户端连接）
    client, addr = server.accept()
    print(f"[+] 连接来自 {addr}")

    # 5. 收发数据
    data = client.recv(1024)
    if data:
        client.sendall(b"Echo: " + data)

    client.close()

C++ 开发者必读：SO_REUSEADDR

这个选项和 C++ 中的行为完全一样——允许重用处于 TIME_WAIT 状态的端口。没有它，关闭服务器后可能要好几分钟才能重新绑定同一端口。

accept() 返回 (conn, address) 元组。这和 C++ 不同：C++ 中 accept() 只返回 fd，地址通过指针参数传出。Python 打包成元组更直观。

实战4. UDP 通信

UDP 是无连接协议——不需要 connect/accept，直接 sendto()/recvfrom()。适合日志收集、DNS 查询、在线游戏等场景。

Python — UDP 发送与接收
import socket

# --- UDP 发送端 ---
sender = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
sender.sendto(b"Hello UDP", ("127.0.0.1", 9999))
sender.close()

# --- UDP 接收端 ---
receiver = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
receiver.bind(("0.0.0.0", 9999))
data, addr = receiver.recvfrom(1024)
print(f"收到来自 {addr}: {data.decode()}")
特性TCP (SOCK_STREAM)UDP (SOCK_DGRAM)
连接面向连接（需三次握手）无连接
可靠性可靠（重传/排序）不可靠（可能丢包）
顺序保证顺序不保证顺序
边界流式（无消息边界）保留消息边界
速度较慢（握手+确认）快
适用场景HTTP、文件传输、SSHDNS、视频流、游戏

工具5. 地址解析

Python 内置的 socket.gethostbyname() 可以在不写 C 代码的情况下做 DNS 解析。对 C++ 开发者来说，这意味着你可以随手写个小脚本验证 DNS 配置。

Python — DNS 解析示例
import socket

# 域名 → IP
ip = socket.gethostbyname("github.com")
print(f"github.com → {ip}")

# IP → 域名（反向解析）
host = socket.gethostbyaddr("8.8.8.8")
print(f"8.8.8.8 → {host[0]}")

# 获取本机主机名
print(f"本机: {socket.gethostname()}")

# 获取本机所有 IP
info = socket.getaddrinfo("localhost", None)
for item in info:
    print(f"  {item[4]}")  # (host, port) 元组
命令行 — 一行 DNS 查询
$ python -c "import socket; print(socket.gethostbyname('github.com'))"
20.205.243.166

$ python -c "import socket; print(socket.getaddrinfo('localhost',80))"
[(AddressFamily.AF_INET, ...), (AddressFamily.AF_INET6, ...)]
C++（getaddrinfo）
struct addrinfo hints{}, *res;
hints.ai_family = AF_UNSPEC;
hints.ai_socktype = SOCK_STREAM;
getaddrinfo("github.com", "80", &hints, &res);
// 遍历 res→ai_next 链表...
// 别忘了 freeaddrinfo(res);
Python
results = socket.getaddrinfo(
    "github.com", 80,
    type=socket.SOCK_STREAM
)
for fam, typ, proto, canon, addr in results:
    print(addr)
# 不需要手动释放

实战6. 端口扫描原理

端口扫描的本质就是尝试连接目标主机的每个端口，根据连接结果判断端口是否开放。

Python — 最简端口扫描（connect 探测）
import socket

def scan_port(host, port, timeout=1.0):
    """尝试连接指定端口，返回 (端口, 是否开放)"""
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.settimeout(timeout)
    try:
        result = sock.connect_ex((host, port))
        return port, result == 0  # connect_ex 返回 0 表示成功
    finally:
        sock.close()

# 扫描 80-85 端口
for port in range(80, 86):
    p, ok = scan_port("scanme.nmap.org", port)
    print(f"Port {p}: {'OPEN' if ok else 'closed'}")
connect_ex() 是 Python 独有的——它和 connect() 一样，但连接失败时不抛异常，而是返回错误码。对于端口扫描来说，这比 try/except 优雅得多。

进阶7. select — 同时处理多个连接

上面的服务器只能串行处理连接（一次一个）。用 select.select() 可以同时监控多个 socket。

Python — select 多路复用
import socket
import select

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
server.bind(("0.0.0.0", 8888))
server.listen(5)
server.setblocking(False)  # 非阻塞模式

sockets = [server]         # 待监控的 socket 列表

while True:
    readable, _, _ = select.select(sockets, [], [], 1.0)
    for s in readable:
        if s is server:
            client, addr = s.accept()
            print(f"[+] {addr}")
            client.setblocking(False)
            sockets.append(client)
        else:
            data = s.recv(1024)
            if data:
                s.sendall(b"Echo: " + data)
            else:
                sockets.remove(s)
                s.close()
C++ 对比
// C++ 中同样使用 select/poll/epoll
// select 的 fd_set 操作很繁琐
fd_set readfds;
FD_ZERO(&readfds);
FD_SET(server_fd, &readfds);
int max_fd = server_fd;
int n = select(max_fd + 1,
              &readfds, NULL, NULL, &tv);
// 还需要手动维护 fd_set 和 max_fd
// 每次 select 后重新设置 fd_set

    sockets, [], [], 1.0
)

        conn, addr = s.accept()
        sockets.append(conn)

        data = s.recv(1024)

Python 3.4+ 推荐用 selectors 模块代替 select——它会自动选择系统最优的多路复用机制（Linux 用 epoll，Windows 用 IOCP，macOS 用 kqueue）。
settimeout 妙用：socket 连接、接收都可设置超时。对于端口扫描，设置连接超时是性能关键——没有超时的话，每个失败端口会卡几十秒！sock.settimeout(0.5) 可以在速度和准确度之间取得不错的平衡。

对比8. Python vs C++ Socket 总结

维度PythonC++
Socket 创建 socket.socket() socket(AF_INET, SOCK_STREAM, 0)
地址结构 ("host", port) 元组 struct sockaddr_in + htons/pton
连接探测 connect_ex() 不抛异常 connect() 返回 -1，检查 errno
字节序 自动处理（内部 htons） 手动 htons/htonl/ntohs/ntohl
资源释放 自动 GC / with 上下文 手动 close() / RAII
IO 复用 selectors 自动选最优 手动选择 select/poll/epoll
错误处理 异常（OSError） 返回值 + errno
多线程安全 GIL 保护但可能影响并发 无锁，手动线程安全设计

规范9. 安全与最佳实践

端口扫描法律风险：未经授权扫描他人主机可能违法。本教程仅用于学习目的和授权安全测试。始终在 localhost 或拥有授权的目标上练习。
