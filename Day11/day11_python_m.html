1. python -m http.server文件传输神器

局域网传文件、本地调试 HTML、CI 分享构建产物——这一条命令解决 80% 的临时分享需求。C++ 要做同样的事需要搭建 Boost.Beast 服务器，Python 一行搞定。

终端命令
# 最基本：8000 端口，共享当前目录
python -m http.server

# 指定端口
python -m http.server 8080

# 指定目录（Python 3.7+）
python -m http.server 3000 -d /path/to/share

# 只允许本机访问（更安全）
python -m http.server 8000 --bind 127.0.0.1

# 远程访问：同事在浏览器输入
# http://192.168.1.100:8000
C++ 实现 HTTP 服务
// 需要 cpp-httplib 第三方库
// CMakeLists.txt 配置、编译...
#include "httplib.h"

int main() {
  httplib::Server svr;
  svr.set_mount_point("/", ".");
  svr.listen("0.0.0.0", 8080);
}

// 还需手动安装库、写 CMake...
Python 一行搞定
# 命令行：直接用，不写代码
# python -m http.server 8080

# 或者在代码中启动：
from http.server import (
    HTTPServer,
    SimpleHTTPRequestHandler
)
server = HTTPServer(
    ('0.0.0.0', 8080),
    SimpleHTTPRequestHandler
)
server.serve_forever()
# 标准库，零依赖

如何快速获取本机 IP（告知同事）

PYTHON
import socket

# 方法1：连接外网选路（最可靠）

s.connect(("8.8.8.8", 80))

# python -c "import socket; s=socket.socket(); s.connect(('8.8.8.8',80)); print(s.getsockname()[0])"
配套工具：本 Day 提供 quick_server.py，比 -m http.server 更强——自动显示 IP、支持文件上传、深色主题目录页、彩色访问日志、CORS 支持。

2. python -m json.toolJSON 格式化&校验

C++ 项目中到处是 JSON：CMake preset、编译数据库 compile_commands.json、REST API 返回值。json.tool 是最快的格式化+校验方式，不需要装 jq。

终端命令
# 格式化 JSON 文件
python -m json.tool compile_commands.json

# 格式化并保存
python -m json.tool ugly.json pretty.json

# 管道输入（curl 返回值格式化）
curl -s https://api.example.com/status | python -m json.tool

# 紧凑输出（Python 3.9+）
python -m json.tool --compact data.json

# 排序键名
python -m json.tool --sort-keys data.json

# 输出无效 JSON 时会报错
echo "{bad json}" | python -m json.tool
# → json.decoder.JSONDecodeError: Expecting property name...
PYTHON 代码等效实现
import json, sys

# 读取 → 格式化输出
data = json.loads(sys.stdin.read())
print(json.dumps(data, indent=2, ensure_ascii=False))

# 常用选项：
# indent=2 / 4   → 缩进
# sort_keys=True → 键名排序
# ensure_ascii=False → 中文不转义
C++ 开发中的具体用途：compile_commands.json 是 Clang 工具链分析项目的基础文件，用这个命令可以快速检查 CMake 生成的内容是否正确，或者提取指定文件的编译选项。

3. python -m zipfile无需安装的解压缩

终端命令
# 列出 ZIP 内容
python -m zipfile -l archive.zip

# 解压到目录
python -m zipfile -e archive.zip ./output/

# 创建压缩包
python -m zipfile -c output.zip file1.cpp file2.h include/

# 测试完整性
python -m zipfile -t archive.zip
PYTHON 代码 — 更灵活的压缩控制
import zipfile, pathlib

# 创建压缩包（代码方式，可精确控制）
with zipfile.ZipFile("release.zip", "w", zipfile.ZIP_DEFLATED) as zf:
    # 直接写入字符串（不需要先创建文件）
    zf.writestr("README.md", "# My Project\n")
    
    # 批量添加 include/ 目录
    for h in pathlib.Path("include").rglob("*.h"):
        zf.write(h)  # 保留相对路径结构

# 解压时按需过滤
with zipfile.ZipFile("archive.zip", "r") as zf:
    # 只解压头文件
    headers = [n for n in zf.namelist() if n.endswith(".h")]
    zf.extractall("./include", members=headers)
对比 C++ 的思维：zf.writestr() 可以直接写入内存字符串，不需要先把文件写到磁盘——这在生成分发包时非常方便（生成 CMakeLists.txt + 源码 → 打包 → 一步到位）。

4. python -m base64编码解码

终端命令
# 编码（输出到终端）
echo -n "user:password" | python -m base64

# 解码（-d 标志）
echo "dXNlcjpwYXNzd29yZA==" | python -m base64 -d

# 编码文件
python -m base64 -e binary_key.bin

# 解码文件
python -m base64 -d encoded.txt > output.bin
PYTHON 代码
import base64

# 标准 Base64
encoded = base64.b64encode(b"C++ Binary Data").decode("ascii")
decoded = base64.b64decode(encoded)

# URL-safe Base64（用于 URL/文件名，无 +/ 字符）
url_encoded = base64.urlsafe_b64encode(b"data")

# HTTP Basic Auth 头（C++ REST 客户端常用）
creds = base64.b64encode(b"admin:secret").decode()
auth_header = f"Authorization: Basic {creds}"

# 内嵌图片到 HTML（data URI）
img_b64 = base64.b64encode(pathlib.Path("logo.png").read_bytes()).decode()
data_uri = f"data:image/png;base64,{img_b64}"

5. python -m hashlib文件哈希校验

下载 C++ SDK、第三方库、预编译二进制时，官网通常会给出 MD5/SHA256。Python 一行校验。

终端命令（Python 3.9+）
# SHA256（推荐）
python -m hashlib -a sha256 boost_1_85_0.tar.bz2

# MD5
python -m hashlib -a md5 file.bin

# 对比官网公布的哈希
python -m hashlib -a sha256 file.bin
# → sha256: e3b0c44298fc1c149...（与官网对比）
PYTHON 代码 — 所有版本可用
import hashlib, pathlib

def verify_file(filepath: str, expected_sha256: str) -> bool:
    """校验文件 SHA256（下载完整性验证）"""
    h = hashlib.sha256()
    # 分块读取，避免大文件占满内存
    with open(filepath, "rb") as f:
        for chunk in iter(lambda: f.read(65536), b""):
            h.update(chunk)
    actual = h.hexdigest()
    ok = actual == expected_sha256.lower()
    print(f"{'✅' if ok else '❌'} {'OK' if ok else 'MISMATCH'}")
    print(f"  期望：{expected_sha256}")
    print(f"  实际：{actual}")
    return ok

# 使用：
# verify_file("boost.tar.gz", "官网公布的sha256")

# 快速单行哈希（python -c）
# python -c "import hashlib; print(hashlib.sha256(open('f','rb').read()).hexdigest())"

6. struct 模块二进制数据解析

C++ 开发必然遇到二进制协议：网络包、文件格式头、序列化数据。Python struct 模块比 C++ 的 reinterpret_cast 更安全，不会越界。

C++ 解析二进制头
// 结构体直接 cast（危险）
struct Header {
    char     magic[4];
    uint16_t version;
    uint8_t  flags;
    uint32_t size;
};
auto* hdr = reinterpret_cast
    <Header*>(buffer);
// 需要注意字节序、对齐...
Python struct 安全解析
import struct

fmt = '>4sHBI'
# > = big-endian
# 4s = char[4]
# H = uint16, B = uint8
# I = uint32
result = struct.unpack(fmt, buf)
magic, ver, flags, size = result
# 自动处理字节序
PYTHON — 常用格式字符
# 字节序前缀
'>'  # big-endian（网络协议）
'<'  # little-endian（x86/x64）
'!'  # network（= big-endian）
'@'  # native（本机默认）

# 基本类型
'b'/'B'  # int8 / uint8   (1 byte)
'h'/'H'  # int16 / uint16 (2 bytes)
'i'/'I'  # int32 / uint32 (4 bytes)
'q'/'Q'  # int64 / uint64 (8 bytes)
'f'     # float32        (4 bytes)
'd'     # float64        (8 bytes)
'4s'    # char[4]        (4 bytes)
'x'     # 填充字节（跳过）

# 实用：解析 PNG 文件头（前8字节）
with open("image.png", "rb") as f:
    sig = struct.unpack('8B', f.read(8))
    is_png = sig[:4] == (137, 80, 78, 71)
    print("✅ PNG" if is_png else "❌ 非PNG")

7. python -c一行代码技巧

不想写脚本文件时的终极武器，C++ 开发者几乎每天都会用到。

日常高频一行代码
# 快速数学计算（比 bc 好用）
python -c "print(2**32)"
python -c "print(0xFF & 0xAB)"
python -c "print(1024*1024*1024, 'bytes =', 1024**3 // 10**6, 'MB')"

# 获取本机 IP
python -c "import socket; s=socket.socket(); s.connect(('8.8.8.8',80)); print(s.getsockname()[0])"

# 端口检测
python -c "import socket; r=socket.socket().connect_ex(('localhost',8080)); print('open' if r==0 else 'closed')"

# 当前时间戳
python -c "import time; print(int(time.time()))"

# 文件 SHA256（所有版本）
python -c "import hashlib; print(hashlib.sha256(open('file','rb').read()).hexdigest())"

# 十六进制 ↔ 十进制
python -c "print(hex(255))"
python -c "print(int('0xff', 16))"

# 格式化 JSON 字符串
python -c "import json,sys; print(json.dumps(json.loads(sys.stdin.read()),indent=2))"

# 查找端口占用
python -c "import socket; [print(p, socket.socket().connect_ex(('localhost',p))) for p in [8000,8080,3000,5000]]"

# URL 编解码
python -c "from urllib.parse import quote; print(quote('C++ /path?a=1&b=2'))"
python -c "from urllib.parse import unquote; print(unquote('C%2B%2B%20test'))"

# 生成随机密码
python -c "import secrets,string; print(secrets.token_urlsafe(16))"

# UUID 生成
python -c "import uuid; print(uuid.uuid4())"
Windows 上注意引号：Windows cmd 用双引号，PowerShell 用单引号或转义。建议在 Git Bash 或 WSL 中使用，引号行为与 Linux 一致。

8. python -m timeit & profile性能分析

timeit — 微基准测试
# 测量表达式执行时间（自动重复 N 次取最小值）
python -m timeit "'-'.join(str(n) for n in range(100))"
# → 3.85 usec per loop

# 带 setup
python -m timeit -s "import re" "re.compile('(\d+)')"

# 指定重复次数
python -m timeit -n 10000 "sum(range(1000))"

# 对比两种写法
python -m timeit "[x**2 for x in range(1000)]"
python -m timeit "list(map(lambda x: x**2, range(1000)))"
profile — 函数调用分析
# 运行脚本并分析性能热点
python -m cProfile script.py
python -m cProfile -s cumulative script.py  # 按累计时间排序
python -m cProfile -o output.prof script.py # 保存到文件

# 分析 pstats
python -c "import pstats; p=pstats.Stats('output.prof'); p.sort_stats('cumulative'); p.print_stats(10)"
C++ 计时方式
// std::chrono
auto start = std::chrono::
  high_resolution_clock::now();

// ... 被测代码 ...

auto end = std::chrono::
  high_resolution_clock::now();
auto us = std::chrono::
  duration_cast<microseconds>(
    end - start).count();
std::cout << us << " us\n";
Python 计时方式
import timeit

# 一行搞定
t = timeit.timeit(
    stmt="sum(range(1000))",
    number=10000
) / 10000
print(f"{t*1e6:.2f} us")

# 或命令行
# python -m timeit "sum(range(1000))"

pdb 基本用法

python -m pdb script.py

# 在代码中设置断点（Python 3.7+）

pdb 常用命令
l / ll     → 列出当前代码（list / longlist）
n          → 执行下一行（step over）
s          → 进入函数（step into）
c          → 继续运行（continue）
q          → 退出调试
p var      → 打印变量值
pp var     → 格式化打印
b 42       → 在第42行设置断点
b func     → 在函数入口设置断点
w          → 查看调用栈（where）
u / d      → 上/下一帧（up/down stack frame）
!expr      → 执行任意 Python 表达式
