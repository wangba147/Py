为什么用 Python 做 C++ 项目自动化

作为 C++ 开发者，你一定遇到过这些痛点：

  • 新增一个 .cpp 文件，需要手动加到 CMakeLists.txt
  • 多个子项目需要逐个编译、依次检查结果
  • 编译错误分散在多个终端窗口，难以汇总
  • 需要在不同配置（Debug/Release/x86/x64）之间反复切换
  • CI 脚本用 bash/bat 写到怀疑人生

Python 天然适合做构建脚本：

能力C++ / Shell 做法Python 做法
遍历文件 find ... -exec 或 std::filesystem pathlib.Path.rglob("*.cpp")
解析文本 写 parser 费劲 / awk/sed 脆弱 re 正则，或手写行扫描
进程调用 system() / fork+exec subprocess.run() 直接拿输出
JSON/YAML 配置 第三方库或手写 标准库 json，或 pyyaml
彩色输出 ANSI 转义码 rich 库开箱即用
HTML 报告 几乎不可能 jinja2 模板引擎
跨平台 Win/macOS/Linux 分别处理 一套代码到处跑
核心理念：Python 不是替代 C++，而是替代 bash/bat 脚本和 手动操作。你的核心逻辑还在 C++ 里，Python 负责编排、检查和汇报。

文件系统操作利器：pathlib + glob

C++17 引入了 std::filesystem，Python 3.4+ 有 pathlib——两者理念极像，都是面向对象的路径操作。

C++ vs Python：文件遍历对比

C++17 filesystem 遍历 .cpp
// C++17: 递归找所有 .cpp 文件
#include <filesystem>
namespace fs = std::filesystem;

for (const auto& entry :
     fs::recursive_directory_iterator("src")) {
    if (entry.path().extension() == ".cpp") {
        std::cout << entry.path() << '\n';
    }
}
Python pathlib 遍历 .cpp
# Python: 递归找所有 .cpp 文件
from pathlib import Path

for f in Path("src").rglob("*.cpp"):
    print(f)

Python 的 rglob 一行搞定，比 C++ 的递归迭代器简洁太多。这正是脚本语言的威力。

C++ 开发者注意：Path.glob() 返回的是生成器（lazy），不是一次性 list。和 C++ ranges 一样的思想。如果需要多次遍历，用 list(Path.rglob(...))。

CMakeLists.txt 解析

解析 CMakeLists.txt 最常见的需求：自动发现源文件、提取目标名、识别依赖库。

核心思路

  1. 逐行读取 CMakeLists.txt
  2. 用正则匹配关键命令：add_executable、add_library、target_sources、target_link_libraries
  3. 处理 CMake 变量替换 ${VAR}
  4. 提取结构化信息

解析 CMakeLists.txt 实战

Python CMakeLists 解析器
import re
from pathlib import Path
from dataclasses import dataclass, field

@dataclass
class CMakeTarget:
    name: str
    target_type: str      # "executable" | "library"
    sources: list = field(default_factory=list)
    links: list = field(default_factory=list)
    includes: list = field(default_factory=list)

def parse_cmake(cmake_path: Path) -> list[CMakeTarget]:
    """解析 CMakeLists.txt，返回所有 target 信息"""
    content = cmake_path.read_text(encoding="utf-8")

    # 收集变量定义，用于替换 ${VAR}
    variables = {}
    for m in re.finditer(r'set\s*\(\s*(\w+)\s+(.+?)\s*\)', content, re.I):
        variables[m.group(1)] = m.group(2).strip()

    def subst(s: str) -> str:
        """替换 ${VAR} 为实际值"""
        def repl(m):
            return variables.get(m.group(1), m.group(0))
        return re.sub(r'\$\{(\w+)\}', repl, s)

    targets = []
    # 匹配 add_executable(名称 source1 source2 ...)
    pattern = r'(add_executable|add_library)\s*\(\s*(\w+)\s+(.*?)\)'
    for m in re.finditer(pattern, content, re.DOTALL | re.I):
        cmd, name, src_block = m.group(1), m.group(2), m.group(3)
        # 按空白字符拆分源文件列表（忽略换行）
        sources = [s for s in re.split(r'\s+', src_block.strip()) if s]
        target_type = "executable" if "executable" in cmd else "library"
        targets.append(CMakeTarget(name=name, target_type=target_type, sources=sources))

    # 提取 target_link_libraries
    link_re = r'target_link_libraries\s*\(\s*(\w+)\s+(.*?)\)'
    for m in re.finditer(link_re, content, re.DOTALL | re.I):
        target_name, libs_str = m.group(1), m.group(2)
        libs = [l.strip() for l in re.split(r'\s+', libs_str.strip()) if l]
        for t in targets:
            if t.name == target_name:
                # 删除 PUBLIC/PRIVATE/INTERFACE 关键字
                t.links = [l for l in libs
                           if l.upper() not in ("PUBLIC","PRIVATE","INTERFACE")]

    return targets


# 使用示例
if __name__ == "__main__":
    cmake_file = Path("CMakeLists.txt")
    if cmake_file.exists():
        for target in parse_cmake(cmake_file):
            print(f"[{target.target_type}] {target.name}")
            print(f"  Sources: {len(target.sources)} files")
            print(f"  Links:   {target.links}")
            print()
局限性：这个简单解析器只能处理 set() / add_executable() / add_library() / target_link_libraries() 的基本用法。复杂的 CMake（条件判断、foreach 循环、函数调用）需要更正经的解析方案（见下文）。

更强大的方案：用 CMake 自身输出信息

与其费劲解析 CMakeLists.txt，不如让 CMake 自己吐出你想要的数据：

cmake --build build --target help

# 获取编译命令（compile_commands.json）

# 用 cmake-file-api 查询详细信息
cmake -B build
# 然后读取 build/.cmake/api/v1/reply/ 下的 JSON 文件

Python 读取 compile_commands.json 可以拿到每个文件的完整编译命令：

import json
from pathlib import Path

def load_compile_commands(build_dir: Path) -> list[dict]:
    cc_path = build_dir / "compile_commands.json"
    if not cc_path.exists():
        raise FileNotFoundError(f"请先配置 cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON")
    return json.loads(cc_path.read_text())

# 获取每个文件的编译命令
commands = load_compile_commands(Path("build"))
for entry in commands[:3]:
    print(f"文件: {entry['file']}")
    print(f"目录: {entry['directory']}")
    print(f"命令: {entry['command'][:100]}...")
    print()
最佳实践：自动化工具优先读 compile_commands.json 而不是解析 CMakeLists.txt。数据更准确、更完整，而且是标准格式。

CMakeLists.txt 自动生成

这才是 Python 自动化的王牌场景——根据项目目录结构自动生成 CMakeLists.txt。

典型场景

  • 新模块开发：在 src/ 下新建目录，自动生成 CMakeLists.txt
  • 骨架代码生成：一键生成 CMake 标准项目结构
  • 多子项目管理：根目录自动扫描所有子目录并 add_subdirectory

自动扫描源文件生成 CMakeLists

Python 生成 CMakeLists.txt
from pathlib import Path
from textwrap import dedent

def generate_cmake(project_name: str,
                      src_dir: Path,
                      output: Path,
                      cpp_std: str = "17",
                      deps: list[str] | None = None):
    """
    根据 src_dir 下的源文件自动生成 CMakeLists.txt

    Args:
        project_name: 项目名
        src_dir: 源代码目录
        output: 输出 CMakeLists.txt 路径
        cpp_std: C++ 标准版本
        deps: 依赖库列表（如 ["Boost::filesystem", "fmt::fmt"]）
    """
    if deps is None:
        deps = []

    # 1. 扫描源文件
    cpp_files = sorted(src_dir.rglob("*.cpp"))
    h_files = sorted(src_dir.rglob("*.h")) + sorted(src_dir.rglob("*.hpp"))

    # 2. 生成文件列表字符串
    sources_str = "\n    ".join(f'"${{CMAKE_CURRENT_SOURCE_DIR}}/{f}"'
                                for f in cpp_files)

    # 3. 找包含目录
    include_dirs = sorted(set(
        d for f in h_files for d in [f.parent]
    ))
    includes_str = "\n    ".join(f'"${{CMAKE_CURRENT_SOURCE_DIR}}/{d}"'
                                for d in include_dirs)

    # 4. 拼装 CMakeLists.txt
    find_packages = ""
    for dep in deps:
        pkg = dep.split("::")[0] if "::" in dep else dep
        find_packages += f'find_package({pkg} REQUIRED)\n'

    links_str = "\n        ".join(deps) if deps else ""
    link_block = f"""
target_link_libraries({project_name} PRIVATE
        {links_str}
)""" if deps else ""

    cmake_content = dedent(f"""\
    cmake_minimum_required(VERSION 3.16)
    project({project_name} LANGUAGES CXX)

    set(CMAKE_CXX_STANDARD {cpp_std})
    set(CMAKE_CXX_STANDARD_REQUIRED ON)

    {find_packages}
    add_executable({project_name}
        {sources_str}
    )

    target_include_directories({project_name} PRIVATE
        {includes_str}
    )
    {link_block.strip()}
    """)

    output.parent.mkdir(parents=True, exist_ok=True)
    output.write_text(cmake_content, encoding="utf-8")
    return len(cpp_files), len(h_files)


# 使用示例
count_cpp, count_h = generate_cmake(
    project_name="MyApp",
    src_dir=Path("src"),
    output=Path("CMakeLists.txt"),
    deps=["fmt::fmt", "spdlog::spdlog"]
)
print(f"生成 CMakeLists.txt: {count_cpp} 个 .cpp, {count_h} 个 .h")
注意：自动生成的 CMakeLists.txt 需要人工检查！它无法推断编译选项、平台差异、第三方库版本等复杂配置。把它当作 90% 的模板，剩下 10% 手动调整。

批量编译：subprocess 实战

回顾 Day 07 的 subprocess，现在把它用到真正的批量构建场景。

单个项目的 CMake 构建流程

import subprocess
import shlex
from pathlib import Path

def build_cmake_project(
    src_dir: Path,
    build_dir: Path,
    build_type: str = "Release",
    generator: str | None = None,
    parallel: int | None = None
) -> tuple[bool, str, str]:
    """
    构建单个 CMake 项目

    Returns:
        (success, stdout, stderr)
    """
    build_dir.mkdir(parents=True, exist_ok=True)

    # —— 阶段 1: 配置 ——
    cmake_config = ["cmake", "-S", str(src_dir),
                    "-B", str(build_dir),
                    f"-DCMAKE_BUILD_TYPE={build_type}"]
    if generator:
        cmake_config.extend(["-G", generator])

    r1 = subprocess.run(cmake_config, capture_output=True, text=True)
    if r1.returncode != 0:
        return False, r1.stdout, f"[CONFIGURE FAILED]\n{r1.stderr}"

    # —— 阶段 2: 构建 ——
    cmake_build = ["cmake", "--build", str(build_dir),
                   "--config", build_type]
    if parallel:
        cmake_build.extend(["--parallel", str(parallel)])

    r2 = subprocess.run(cmake_build, capture_output=True, text=True)
    if r2.returncode != 0:
        return False, r2.stdout, f"[BUILD FAILED]\n{r2.stderr}"

    return True, r2.stdout, ""
💡 对比 make：如果不用 cmake --build，可以直接调用 make/ninja/msbuild。CMake 的好处是 --build 抽象了底层构建工具。

批量构建多个子项目

from dataclasses import dataclass
from concurrent.futures import ThreadPoolExecutor, as_completed

@dataclass
class BuildResult:
    project: str
    success: bool
    duration: float
    errors: str = ""

def batch_build(projects: dict[str, Path],
                 output_root: Path,
                 build_type: str = "Release",
                 max_workers: int = 4) -> list[BuildResult]:
    """
    批量构建多个 CMake 项目

    Args:
        projects: {项目名: 源码目录}
        output_root: 构建输出根目录
        max_workers: 并行构建线程数（注意内存！）
    """
    import time
    results = []

    def build_one(name, src):
        t0 = time.time()
        ok, out, err = build_cmake_project(
            src, output_root / name, build_type
        )
        elapsed = time.time() - t0
        return BuildResult(name, ok, elapsed, err)

    with ThreadPoolExecutor(max_workers=max_workers) as pool:
        futures = {
            pool.submit(build_one, name, src): name
            for name, src in projects.items()
        }
        for future in as_completed(futures):
            result = future.result()
            results.append(result)
            status = "OK" if result.success else "FAILED"
            print(f"[{status}] {result.project} ({result.duration:.1f}s)")
            if result.errors:
                print(result.errors[:200])

    return results
并行编译须知：多个 CMake 项目并行编译会同时消耗大量 CPU 和内存。max_workers 设 2~4 比较安全。每个项目内部的 --parallel 用 CMake 自己控制。

C++ vs Python 构建脚本：直观对比

依赖检测与增量构建

增量构建：只重新编译修改过的文件。CMake + Ninja/MSBuild 自动处理，但我们也可以用 Python 做预检查——比如只对有变更的子项目运行构建。

检测修改过的源文件

import os
from pathlib import Path

def find_changed_sources(src_dir: Path, last_build_time: float) -> list[Path]:
    """
    返回自上次构建以来修改过的源文件

    Args:
        src_dir: 源码目录
        last_build_time: 上次构建的 Unix 时间戳
    """
    changed = []
    for f in src_dir.rglob("*"):
        if f.is_file() and f.suffix in (".cpp", ".h", ".hpp", ".c"):
            if f.stat().st_mtime > last_build_time:
                changed.append(f)
    return changed


def should_rebuild(src_dir: Path, build_marker: Path) -> bool:
    """
    判断是否需要重新构建

    策略：检查 src_dir 下是否有比 build_marker 更新的文件
    """
    if not build_marker.exists():
        return True  # 从未构建过

    last_build = build_marker.stat().st_mtime
    for ext in ("*.cpp", "*.h", "*.hpp", "*.c", "CMakeLists.txt"):
        for f in src_dir.rglob(ext):
            if f.stat().st_mtime > last_build:
                print(f"  → 检测到变更: {f.name}")
                return True
    return False


# 用法：在构建成功后写一个时间戳文件
def mark_build_complete(marker: Path):
    marker.write_text("")  # touch 文件

# 构建流程
build_marker = Path("build/.last_build")
if should_rebuild(Path("src"), build_marker):
    print("需要重新编译...")
    success, _, _ = build_cmake_project(Path("."), Path("build"))
    if success:
        mark_build_complete(build_marker)
        print("构建成功！")
进阶：真实项目用 git diff --name-only HEAD 判断变更更准确。可以用 subprocess 调用 git 命令，比文件时间戳更可靠。

Python 读取配置驱动构建

import json
from pathlib import Path

class BuildConfig:
    def __init__(self, config_path: Path):
        data = json.loads(config_path.read_text())
        self.workspace = Path(data["workspace"])
        self.build_root = Path(data["build_root"])
        self.build_type = data.get("build_type", "Release")
        self.generator = data.get("generator")
        self.projects = data["projects"]

    def get_enabled_projects(self):
        """返回 {项目名: 源码路径}，跳过 skip=true 的项目"""
        return {
            p["name"]: self.workspace / p["src"]
            for p in self.projects
            if not p.get("skip", False)
        }


# 驱动构建
config = BuildConfig(Path("build_config.json"))
projects = config.get_enabled_projects()
print(f"将构建 {len(projects)} 个项目")

results = batch_build(
    projects=projects,
    output_root=config.build_root,
    build_type=config.build_type
)
YAML 也推荐使用：json 是标准库，零依赖；但 YAML 可读性更好、支持注释。如果项目已有 pyyaml，可以轻松切换：import yaml; data = yaml.safe_load(config_path.read_text())

代码生成：从模板到 .h/.cpp

还记得 Day 04 的 cpp_class_generator.py 吗？这里是升级版——基于模板引擎批量生成项目文件。

用 Jinja2 生成头文件

# pip install jinja2
from jinja2 import Template
from pathlib import Path

# 模板（可以放在单独文件）
HEADER_TEMPLATE = """\
/**
 * @file {{ filename }}.h
 * @brief {{ brief }}
 * Auto-generated by build script — DO NOT EDIT manually.
 */
#pragma once

{%- for inc in includes %}
#include <{{ inc }}>
{%- endfor %}

namespace {{ namespace }} {

{%- for method in methods %}
    {{ method.signature }};
{%- endfor %}

} // namespace {{ namespace }}
"""

def generate_header(config: dict, output_dir: Path):
    """从配置生成 .h 文件"""
    template = Template(HEADER_TEMPLATE)
    content = template.render(**config)
    output = output_dir / f"{config['filename']}.h"
    output.parent.mkdir(parents=True, exist_ok=True)
    output.write_text(content, encoding="utf-8")
    return output


# 批量生成接口文件
api_defs = [
    {
        "filename": "ILogger",
        "brief": "日志接口",
        "namespace": "myapp",
        "includes": ["string", "string_view", "memory"],
        "methods": [
            {"signature": "virtual void log(std::string_view msg) = 0"},
            {"signature": "virtual void error(std::string_view msg) = 0"},
            {"signature": "virtual ~ILogger() = default"},
        ]
    },
    {
        "filename": "ISerializer",
        "brief": "序列化接口",
        "namespace": "myapp",
        "includes": ["string", "vector", "cstdint"],
        "methods": [
            {"signature": "virtual std::string serialize() const = 0"},
            {"signature": "virtual bool deserialize(std::string_view data) = 0"},
        ]
    },
]

for api in api_defs:
    out = generate_header(api, Path("generated"))
    print(f"生成: {out}")

测试运行与结果聚合

构建完成后自动跑测试、汇总结果。

自动发现并运行 CTest / gtest 可执行文件

import subprocess
import re
from pathlib import Path

def run_ctest(build_dir: Path, timeout: int = 60) -> dict:
    """
    在构建目录运行 CTest，解析结果

    Returns:
        {"total": int, "passed": int, "failed": int, "failures": [...]}
    """
    r = subprocess.run(
        ["ctest", "--test-dir", str(build_dir),
         "--output-on-failure"],
        capture_output=True, text=True,
        timeout=timeout
    )
    output = r.stdout + r.stderr

    # 解析 CTest 输出
    total = 0; passed = 0; failed = 0; failures = []
    for line in output.splitlines():
        if "tests from" in line:
            m = re.search(r'(\d+) tests from', line)
            if m: total += int(m.group(1))
        if "100% tests passed" in line:
            m = re.search(r'(\d+) tests passed', line)
            if m: passed += int(m.group(1))
        if "FAILED" in line and "test" in line.lower():
            failures.append(line.strip())

    failed = total - passed if total > 0 else 0
    return {
        "total": total, "passed": passed,
        "failed": failed, "failures": failures
    }


def find_and_run_gtest(build_dir: Path, pattern: str = "*test*") -> list[dict]:
    """
    在构建目录找 gtest 可执行文件并逐个运行

    Returns:
        [{name, success, output, ...}]
    """
    results = []
    for exe in build_dir.rglob(pattern):
        if exe.is_file() and os.access(exe, os.X_OK):
            r = subprocess.run(
                [str(exe), "--gtest_brief=1"],
                capture_output=True, text=True, timeout=30
            )
            results.append({
                "name": exe.name,
                "success": r.returncode == 0,
                "output": r.stdout,
                "errors": r.stderr
            })
    return results
